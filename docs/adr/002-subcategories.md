# Spec: Subcategory Support

## Overview

Extend the flat categories system to support a 2-level hierarchy (Category → Subcategory). Transactions remain assignable to any level. The Dashboard aggregates by top-level category with drill-down into subcategories.

## Design Decisions

| Decision | Choice |
|---|---|
| Hierarchy depth | 2 levels only (Category → Subcategory) |
| Transaction assignment | Any level (top-level OR subcategory) |
| Existing categories | Remain as top-level parents |
| Subcategory color | Inherit parent's color (no separate picker) |
| Default subcategories | Seeded in migration |
| Dashboard view | Top-level aggregation with drill-down |

---

## Default Subcategories (Seeded)

| Parent | Subcategories |
|---|---|
| Food & Dining | Groceries, Restaurants |
| Travel | Flights, Accomodations |
| Transportation | Gas & Charging |

## Phase 1: Database Schema

### Migration `000003_add_subcategory_support`

**Up:**
- Add nullable `parent_id UUID REFERENCES categories(id) ON DELETE CASCADE` to `categories`
- Add index on `parent_id`
- Seed default subcategories (see table above)

**Down:**
- Delete all seeded subcategories (`DELETE WHERE parent_id IS NOT NULL`)
- Drop the `parent_id` column

### Updated SQL Queries (`backend/db/query.sql`)

- Include `parent_id` in all category `SELECT` fields: `GetCategories`, `CreateCategory`, `UpdateCategory`
- Add `GetTopLevelCategories` — `WHERE parent_id IS NULL`
- Add `GetSubcategoriesByParent` — `WHERE parent_id = $1`

Regenerate `backend/db/generated/` via `sqlc generate` after changes.

---

## Phase 2: Backend API

### Model Changes (`backend/models.go`)

Add to `Category` struct:
- `ParentID *string`
- `Subcategories []Category`

### Handler Changes (`backend/categories.go`)

| Endpoint | Change |
|---|---|
| `GET /api/categories` | Return nested tree: top-level categories with `subcategories` array embedded, built in-memory |
| `POST /api/categories` | Accept optional `parent_id`; validate parent exists and is itself top-level (enforces 2-level max) |
| `PUT /api/categories/:id` | Same depth validation; block re-parenting a category that already has children |
| `DELETE /api/categories/:id` | No change — `ON DELETE CASCADE` at DB level handles subcategory removal |

Update Swagger annotations on all modified handlers. Run `make generate-docs`.

### Test Coverage (`backend/categories_test.go`)

- Create subcategory with valid `parent_id` → succeeds
- Create subcategory with non-existent `parent_id` → 400
- Create subcategory of a subcategory → 400 (depth enforcement)
- `GET /api/categories` → returns nested structure
- Delete top-level category → subcategories are also removed (cascade)

## Phase 3: Frontend — Settings

### Type Changes (`frontend/src/types.ts`)

Add to `Category` interface:
- `parentId?: string`
- `subcategories?: Category[]`

### UI Changes (`frontend/src/Settings.tsx`)

- Top-level category cards gain an expandable subcategory list below the existing card content
- Each top-level card has an "Add Subcategory" button → opens create modal with `parentId` pre-filled
- Subcategory create/edit modal: name + description fields only (no color picker; color inherited from parent)
- Subcategory cards rendered as a sub-list inside the parent card
- Deleting a top-level category shows a warning that its subcategories will also be deleted

---

## Phase 4: Frontend — Dashboard

### Category Selector (`frontend/src/Dashboard.tsx`)

Change flat `Select` dropdown to grouped select using Ant Design `OptGroup`:
- Top-level categories as group labels (also selectable as option values)
- Subcategories appear as indented options within their parent's group

### Pie Charts (`frontend/src/Dashboard.tsx`)

- **Default view**: aggregate by top-level category — transactions assigned to a subcategory roll up to their parent
- **Drill-down**: clicking a top-level slice that has subcategories reveals a second breakdown chart for that category's subcategories
- **Back button** returns to the top-level view
- Transactions assigned directly to a top-level category (with no subcategory) appear only in the top-level chart

### Color Utility (`frontend/src/utils.ts`)

Update `getCategoryColor()`: if the category has a `parentId`, look up and return the parent's color from the full category list.

## Out of Scope

- CSV auto-mapping to subcategories (existing mapping to top-level categories remains)
- Per-subcategory color customization
- More than 2 levels of nesting
